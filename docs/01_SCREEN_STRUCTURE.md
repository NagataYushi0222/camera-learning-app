# Screen Structure

## Basic layout

デスクトップ版の基本画面は以下。

左側：
- 3Dビュー
- 光源、物体、レンズ、ピンホール、スクリーンを表示
- ユーザーは自由に回転・ズーム・移動できる

右側：
- スクリーン / センサー上の実際の像
- ピクセルグリッド表示
- クリックしたピクセルの情報表示
- 現在の明るさ、対応する光線数、モード情報など

下部：
- ノベルゲーム風の解説ボックス
- ガイドキャラクター
- 前へ / 次へ
- 3Dで確かめる
- この章の要点

上部：
- 章タイトル
- 進行状況
- 目次
- 設定
- 表示モード切り替え

## Should explanation and view be separate?

完全に交互にするのではなく、2種類の画面を使い分ける。

### Story / Explanation Screen

説明を中心にする画面。
大きな背景イラストとノベルゲーム風のテキストボックスを表示する。

使う場面：
- 新しい概念の導入
- 「見えるとは何か」などの問いかけ
- 目とカメラの対応関係の説明

### Interactive Simulation Screen

説明と3Dビューとセンサー表示を同時に見る画面。

使う場面：
- レンズなし
- ピンホール
- レンズあり
- ピント調整
- 絞り
- センサー1ピクセル解析

## Recommended flow

1. ストーリー画面で問いを提示
2. シミュレーション画面で確かめる
3. 右側スクリーンをクリックしてピクセル解析
4. ガイドが意味を説明
5. 次の概念へ進む
