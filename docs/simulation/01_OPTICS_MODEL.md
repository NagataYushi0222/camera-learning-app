# Optics Model

## Policy

このアプリは教育用シミュレーションであり、完全な物理シミュレーターではない。

優先順位は以下。

1. 学習者が本質を理解できること
2. 視覚的に分かりやすいこと
3. 操作が軽いこと
4. 必要な範囲で物理的に正しいこと

## Light model

光は本質的には波として説明する。

ただし、実装上は以下を組み合わせる。

- 波面表示：光が空間に広がることを示す
- 光線表示：波の進行方向を示す補助線
- センサー表示：光の強度と色を2D画像として表示する

## Important explanation

光線は、実際の光が細い線として1本だけ存在するという意味ではない。
光線は、波の進行方向を表すための補助表示である。

## Simplification

初期MVPでは、厳密な波動光学、干渉、フーリエ光学などは扱わない。

ただし、以下は説明として入れる。

- ピンホールを小さくしすぎると回折でぼやける
- 絞りすぎると回折で解像感が下がる

初期実装では、回折は簡易的なボケ表現として扱ってよい。

## 3D lens and pinhole model

3Dビューのレンズは、表示用には光軸まわりに回転した双曲面レンズとして生成する。

- 焦点距離が短いほど曲率半径を小さくする
- 曲率半径と口径から双曲線のサグ量を計算する
- 中心厚は「端厚 + 両面のサグ量」で決まり、焦点距離や口径の変化に合わせて変わる

センサー画像の計算自体は薄レンズ式 `1/f = 1/do + 1/di` を使う。
そのため、3D形状は学習者が焦点距離・曲率・厚みの関係を理解するための教育用モデルであり、厚肉レンズの厳密な屈折追跡ではない。

ピンホール板は、表示用にも実際の円形穴を持つ板として生成する。
穴の半径はシミュレーションの `pinholeRadius` と連動する。
