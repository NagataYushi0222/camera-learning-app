# MVP Simplified Optics Model

初期MVPでは、学習フローと応答性を優先するため、光学計算は教育用の簡易レイトレーシングモデルとして実装する。

- 光線は、波の進行方向を説明する補助線として表示する。
- りんごは厳密な3D表面ではなく、赤い本体、茶色い茎、緑の葉を点群サンプルとして扱う。
- 右側センサー画像はCSSの疑似画像ではなく、点群サンプルから飛ばした光線のセンサー到達結果をCanvas 2Dへ描画する。
- 通常のセンサー表示ではCanvas全体に後処理blurをかけず、各rayのセンサー面ヒットをGaussian splatで再構成する。これにより、光線計算から生じる像の広がりを見やすくする。
- サンプル点表示では、点群とray hitの粗さをあえて確認できるようにする。
- レンズなしでは、各物体サンプルからスクリーン全体へ届く直達光を加算し、1ピクセルに多くの物体点が混ざる状態を表す。
- ピンホールでは、ピンホール半径内を通る光だけをセンサーへ進める。光は曲げず、穴で経路を制限する。
- レンズありでは、薄レンズの式 `1/f = 1/do + 1/di` から理想像面を求め、レンズ開口上の複数点を通る光線を理想像点へ向ける。
- ピントずれでは、同じ薄レンズ計算を使い、センサー位置が理想像面から外れた結果として交点が広がる状態を表す。
- レンズのボケ量は、センサー位置と理想像面の差、およびレンズ口径に応じて大きくなる。口径を小さくすると、ピントずれ時の広がりは小さくなる。
- 右側で選択したピクセルに対して左側3Dビューに表示する光線は、センサー描画時にそのピクセルへ寄与した代表光線を使う。
- 3Dビューの光線は、初期状態では代表光線だけを表示し、選択ピクセル解析、光束表示、全光線デバッグを切り替えられる。

この段階では、厳密な波動光学、干渉、回折、レンズ収差、実レンズ厚、遮蔽、物体表面の完全なBRDFは扱わない。
後続フェーズで、各モードの教育目的に合わせて計算精度と表示を段階的に高める。
